## 2.2.1. Amazon EC2 인스턴스
가상의 컴퓨팅 환경으로 CPU, 메모리, 스토리지, 네트워킹 용량을 결정하는 다양한 인스턴스 유형을 제공한다. 즉, 클라우드 환경에서 컴퓨팅 자원을 필요한 만큼 사용하고, 쓰임을 다하면 자원을 반납하는 형태로 임의로 구성된 인스턴스이다.

### 인스턴스 유형
500개가 넘는 인스턴스 유형이 존재하여 사용자 목적과 비즈니스 환경에 맞게 최적화하여 선택하면 된다.
참고로 프리티어는 t2.micro 인스턴스 유형만 제공한다.
[인스턴스 유형 확인하기](https://aws.amazon.com/ko/ec2/instance-types/)

### 인스턴스 상태
![lifecycle](https://datascientest.com/en/files/2023/10/AWS-EC2-schema.jpg){: width="300" height="200" } _인스턴스의 생명주기_

> '중지됨' 상태와 '종료됨' 상태를 명확히 구분해야 한다.
- '중지됨': 인스턴스를 유지한 채 일시적으로 중지한 상태 -> 다시 구동 가능
- '종료됨': 인스턴스를 영구 삭제 -> 다시 사용 불가
또한 '종료됨' 상태가 되어도 관리 콘솔상에서 바로 삭제되지 않고 일정 시간 정보를 출력한다. 종료되지 않은 건 아니므로 다른 작업을 수행해도 된다.

## 2.2.2. AMI(Amazon Machine Image)
인스턴스를 시작할 때 필요한 정보를 제공하는 것으로, OS와 SW를 적절히 구성한 상태로 제공되는 템플릿이다.
